## 一、项目定位

**智瘤方舟 (MediArk)** 是面向肿瘤科临床医生的多模态生成式 AI 患者教育平台。医生录入患者画像后，系统自动调用大语言模型(LLM)、AI 绘图和 AI 视频生成管线，产出个性化科普文案、海报和短视频，经专家审核后发布给患者。

**核心创新点**：
1. 患者画像驱动的个性化内容生成（心理状态、认知水平、接收偏好 → 差异化内容策略）
2. 多模态 AI 管线一键生成（文字 + 海报 + 图片 + 视频）
3. 循证知识库约束 AI 幻觉（NCCN/CSCO 指南作为生成锚点）
4. 专家审核闭环（Human-in-the-loop，医疗内容发布前必经专家审核）

**团队**：薪火承梦

## 二、技术架构

### 2.1 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 后端 | Node.js 22 + Express 5 | 轻量单体，零外部依赖 |
| 数据库 | SQLite (WAL 模式) | 内嵌式，10 张表 + 11 个索引 |
| 前端 | HTML5 + Tailwind CSS + Vanilla JS | 无框架，CDN 引入 |
| AI 文本 | 豆包大模型 (Doubao) | OpenAI 兼容接口 |
| AI 图片 | 字节跳动 Seedream 5.0 | BytePlus ModelArk |
| AI 视频 | 字节跳动 Seedance 1.5 Pro | 异步任务 + 轮询 |
| 部署 | Render (Web Service) | render.yaml 自动部署 |

## 四、AI 生成管线

### 4.1 管线流程

```
患者画像 + 医生医嘱 + 输出格式
          │
          ▼
  ┌───────────────────────┐
  │  1. Master JSON 生成   │ ← 豆包大模型 (Doubao)
  │  临床摘要 / 必须知道   │   OpenAI 兼容 /chat/completions
  │  必须做 / 必须避免     │
  │  危险信号 / 旁白脚本   │
  └───────────┬───────────┘
              │
     ┌────────┼────────┐
     ▼        ▼        ▼
  ┌──────┐ ┌──────┐ ┌──────┐
  │ 海报  │ │ 图片  │ │ 视频  │
  │poster │ │image │ │video │
  │浏览器 │ │Seed- │ │Seed- │
  │渲染   │ │ream  │ │ance  │
  └──────┘ └──────┘ └──┬───┘
                       │ 异步
                       ▼
                  自动轮询(10s×18)
                       │
                       ▼
              ┌────────────────┐
              │  作品保存到数据库  │
              │  status=generated │
              └────────────────┘
```

3. **支撑患者教育** — 生成的科普内容可引用知识库中的证据等级和核心要点

## 八、安全机制

| 机制 | 实现状态 | 说明 |
|------|---------|------|
| 密码哈希 | crypto.scryptSync + 16字节盐 | 抗彩虹表 |
| 时序安全比较 | crypto.timingSafeEqual | 防时序攻击 |
| Session 管理 | SHA256 哈希存储 + 14天过期 | httpOnly cookie |
| SQL 注入防护 | 参数化查询 (better-sqlite3) | 全部端点 |
| 角色权限 | requireAuth + requireRole 中间件 | admin / doctor |
| Cookie 安全 | httpOnly + sameSite=lax | 防 XSS 读取 |

## 十、比赛 Demo 演示流程

1. 打开平台首页 → 了解项目定位与团队
2. 用 `doctor / doctor123` 登录 → Dashboard 显示真实统计 + 图表
3. 进入「患者管理」→ 查看预置的 4 名典型食管癌患者画像
4. 进入「知识库」→ 浏览 15 条循证知识（点击知识图谱节点筛选）
5. 进入「生成科普作品」→ 下拉选择"张建国" → 患者信息自动填充
6. 配置科普重点 + 输出格式 → 点击"开始生成" → AI 实时生成文案+图片+视频
7. 审核内容 → 点击"确认发布"
8. 进入「我的作品」→ 查看作品状态，点击二维码按钮 → 生成患者扫码链接
9. 提交专家审核 → 用 `reviewer / review123` 登录 → "待我审核" Tab → 审核通过/退回
10. 进入「作品库」→ 查看已发布的科普作品

## 十一、已知限制与后续规划

### 当前限制

1. 知识库暂未作为 RAG 上下文注入 AI 生成 prompt（接口已预留）
2. 无分页（列表 API 一次返回全部数据）
3. 无 WebSocket 实时推送（视频生成依赖轮询）
4. 无通知系统（审核结果无推送）
5. 无患者端访问追踪（二维码生成后无法统计阅读率）

### 后续规划

1. RAG 集成 — 知识库作为向量检索上下文注入 LLM prompt
2. 患者端小程序 — 微信扫码查看 + 阅读率追踪
3. 数据库迁移 — SQLite → PostgreSQL
4. 多癌种扩展 — 知识库支持肺癌、胃癌、结直肠癌等
5. 内容模板系统 — 保存成功作品为模板复用
6. 多医院/多科室 — 组织层级与数据隔离
